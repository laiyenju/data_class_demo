# 簡報大綱：旅平險客群分群與回購策略

> 文件目的：更新原本「客群分群結合生存分析與回購策略」的分析方法，納入助教建議，避免提前洩漏答案，並讓期末報告更像正式的資料分析專案。  
> 核心修正：不再使用 2023-2025 完整資料直接做分群；改為用 2023-2024 建立客群，再用 2025 驗證客群回購差異。

## 1. 期末報告應包含的必要內容

參考過去兩份優秀期末簡報後，本次報告建議使用以下架構：

| 報告章節 | 必要內容 | 對本題的作用 |
| --- | --- | --- |
| 商業問題 | 說明旅平險回購與再行銷痛點 | 讓題目不只是做模型，而是回答如何提高回購 |
| 資料與 EDA | 2023-2024 vs 2025 客戶數、回購人數、回購間隔、保費價值 | 說明資料長相與為何需要分群 |
| 方法架構 | 時間切分、特徵設計、K-means、PCA、生存分析 | 讓助教與組員知道分析邏輯可被驗證 |
| 分群結果 | 最終客群數、各群輪廓、客群命名 | 回答「客戶能否被分成不同客群」 |
| 驗證結果 | 2025 回購率、回購速度、回購曲線 | 回答「不同客群回購速度是否不同」 |
| 商業應用 | 各客群提醒時機、提醒內容、文案示範 | 回答「如何設計差異化再行銷策略」 |
| 限制與風險 | 右設限、時間窗差異、非因果、特徵限制 | 避免過度宣稱模型能力 |

本次報告的主軸建議濃縮成一句話：

> 用 2023-2024 的完整投保行為建立旅平險客群，再用 2025 回購行為驗證各群是否具有不同回購速度，最後將結果轉換成差異化提醒時機與文案策略。

## 2. 研究問題與成功標準

本報告要回答三個問題：

1. 旅平險客戶是否能依回購行為、旅遊型態、保費價值與購買習慣分成不同客群？
2. 不同客群在 2025 的回購速度與回購週期是否不同？
3. 各客群是否適合設計不同的再行銷策略？應如何設計？

建議的成功標準：

| 成功標準 | 判斷方式 |
| --- | --- |
| 分群具統計可分性 | Silhouette 較高、Davies-Bouldin 較低、PCA 視覺化有一定分離 |
| 分群具商業可解釋性 | 每群可用 1-2 句話命名與描述 |
| 分群具驗證價值 | 各群在 2025 的回購率或回購時間分布有可解釋差異 |
| 分群能轉成策略 | 每群可提出提醒時間、文案方向與優先級 |

## 3. 資料切分與避免資料洩漏

### 3.1 時間切分

| 資料集 | 日期範圍 | 用途 |
| --- | --- | --- |
| 建模資料集 | 2023-01-01 至 2024-12-31 | 建立客戶層級特徵、跑分群、選 K 值 |
| 驗證資料集 | 2025-01-01 至 2025-12-31 | 驗證各客群是否有不同回購率與回購速度 |

切分日期以 `LOGIN_DATE` 為準，因為本研究關心的是「何時完成投保」這個購買行為。

### 3.2 不可提前洩漏的內容

在建立 2023-2024 客群時，不可使用任何 2025 才知道的資訊，例如：

- 2025 是否回購。
- 2025 第一次投保日期。
- 2025 投保次數。
- 2025 保費。
- 2025 目的地。
- 2025 回購間隔。

2025 只能在分群完成後，用來檢查各群後續表現。

### 3.3 建議簡報圖

建議放一張時間軸：

```text
2023-01-01                         2024-12-31  2025-01-01                         2025-12-31
|---------------------------------------------|  |---------------------------------------------|
                建模資料集                                      驗證資料集
     建立 RFM 與行為特徵、跑 K-means                         驗證回購率與回購速度
```

## 4. 既有方法提案需要修改的地方

| 舊版方法 | 問題 | 修正版 |
| --- | --- | --- |
| 使用 2023-2025 完整行為做分群 | 會把 2025 答案提前放進特徵，無法驗證 | 只用 2023-2024 分群，2025 驗證 |
| 一次購買客獨立排除於 K-means | 舊版是因為回購間隔缺值 | 新版 RFM 可讓一次購買客納入，因為 F=1、R/M 都有值 |
| 特徵過多且含大量比例欄位 | K-means 容易被雜訊干擾，輪廓係數不好 | 先以 RFM 為核心，再逐步加入少量加值特徵 |
| 分群特徵含回購間隔 | 容易造成「用回購速度分群，再證明回購速度不同」 | 回購間隔改放在驗證與策略解讀，不作為核心分群特徵 |
| 生存分析像是在證明分群效果 | 可能被誤解為因果或預測模型 | 生存分析定位為把分群結果轉成提醒時機 |

## 5. 客戶層級特徵設計

### 5.1 基本分析單位

原始資料是一列一筆投保紀錄；分群資料需整理成一列一位客戶。

| 項目 | 口徑 |
| --- | --- |
| 客戶識別 | `APL_ID` |
| 投保日期 | `LOGIN_DATE` |
| 建模期間 | 2023-2024 |
| 驗證期間 | 2025 |
| 分群單位 | 2023-2024 曾投保的客戶 |

### 5.2 RFM 核心特徵

RFM 是本次修正版的核心特徵，原因是簡潔、可解釋、適合行銷策略。

| 特徵 | 計算方式 | 商業意義 |
| --- | --- | --- |
| R：Recency | 2024-12-31 減去該客戶在 2023-2024 的最後一次投保日期 | 距離上次投保多久，越小代表越近期活躍 |
| F：Frequency | 該客戶在 2023-2024 的投保次數 | 投保頻率 |
| M：Monetary | 該客戶在 2023-2024 的累積保費，並可輔以平均保費 | 客戶保費價值 |

建議進模型前處理：

- `F`、累積保費、平均保費可視分布使用 `log1p`。
- 所有數值特徵進 K-means 前使用標準化。
- 保留原始值做客群解釋，標準化值只供模型使用。

### 5.3 加值候選特徵

加值特徵不要一次全部加入，而是分組測試。

| 類型 | 候選特徵 | 加入理由 |
| --- | --- | --- |
| 旅遊型態 | 平均旅遊天數、海外占比、多目的地占比 | 區分短天數、長天數、複雜行程 |
| 購買習慣 | 平均提前投保天數、3 天內投保占比 | 區分臨時投保與提前規劃 |
| 保費價值 | 平均保費、每旅遊日保費 | 區分高價值與輕量投保 |
| 目的地偏好 | 日本/韓國/國內/長程目的地占比 | 僅建議放少數高占比目的地，避免維度過多 |

不建議作為核心分群特徵：

- 大量 `CARD_BANK` 類別或銀行比例。
- 過細目的地 one-hot。
- 2025 行為。
- 回購間隔。

## 6. 特徵組合測試設計

為回應助教建議，建議不要只跑一組特徵，而是測試多組特徵組合。

| 組合 | 特徵內容 | 目的 |
| --- | --- | --- |
| A | RFM only | 建立最簡潔、最可解釋的 baseline |
| B | RFM + 旅遊型態 | 檢查旅遊天數與國內外型態是否改善分群 |
| C | RFM + 購買習慣 | 檢查提前投保行為是否形成差異 |
| D | RFM + 保費價值 | 檢查高價值客是否更清楚被分出 |
| E | RFM + 精簡全特徵 | 在維度仍可控時取得較完整輪廓 |

每組特徵都跑 K=2 到 K=6，並輸出：

| 評估指標 | 用途 |
| --- | --- |
| Silhouette score | 越高越好，衡量群內近、群間遠 |
| Davies-Bouldin index | 越低越好，衡量群間分離 |
| Calinski-Harabasz score | 越高越好，衡量群間變異與群內變異 |
| 最小群體占比 | 避免產生太小、無法行銷操作的群 |
| 商業可解釋性 | 是否能命名、能轉策略 |

最終選擇不只看最高分，而是看「統計表現 + 商業解釋 + 2025 驗證」。

## 7. 分群流程

建議流程如下：

```text
原始投保資料
    ↓
依 LOGIN_DATE 切分 2023-2024 / 2025
    ↓
建立 2023-2024 客戶層級 RFM 與候選特徵
    ↓
特徵轉換與標準化
    ↓
測試 A-E 特徵組合、K=2 到 K=6
    ↓
選出最終特徵組合與 K 值
    ↓
建立最終客群並命名
    ↓
用 2025 驗證回購率與回購速度
    ↓
設計提醒時機與文案
```

## 8. PCA 視覺化

PCA 的定位是「視覺化與輔助判斷」，不是一定要用 PCA 後的三維資料分群。

建議輸出：

1. PCA 3D scatter plot，以分群標籤上色。
2. PCA explained variance ratio，說明前三個主成分可解釋多少變異。
3. 若 PCA 圖完全混在一起，代表特徵區隔力可能不足，需要回頭調整特徵組合。

簡報上的說法：

> PCA 讓我們把多個投保行為特徵壓縮成三維空間觀察，若客群在三維空間中有一定分離，代表所選特徵能有效反映客戶行為差異。

## 9. 2025 驗證設計

### 9.1 驗證對象

以 2023-2024 曾投保並已被分群的客戶為主，觀察他們在 2025 是否再次投保。

### 9.2 回購事件定義

| 項目 | 定義 |
| --- | --- |
| 起始時間 | 該客戶在 2023-2024 的最後一次投保日 |
| 事件 | 2025 年內第一次再次投保 |
| 事件時間 | 2025 第一次投保日 - 2023-2024 最後一次投保日 |
| 未回購者 | 右設限至 2025-12-31 |

### 9.3 驗證指標

| 指標 | 用途 |
| --- | --- |
| 2025 回購率 | 各群是否有不同回購比例 |
| 回購天數平均/中位數 | 各群回購速度快慢 |
| 30/60/90/180/365 天累積回購率 | 轉換成提醒策略 |
| Kaplan-Meier 曲線 | 比較各群回購速度 |

建議加入一句限制：

> 2025 驗證仍是觀察性資料，不代表分群造成回購差異；它的用途是檢查客群是否具備可行銷操作的回購時間差異。

## 10. 生存分析定位

生存分析不應被包裝成因果證明，而是策略轉換工具。

| 生存分析輸出 | 商業用途 |
| --- | --- |
| 各群 60 天回購率 | 判斷是否適合短期提醒 |
| 各群 120/180 天回購率 | 判斷中期提醒窗口 |
| 各群 365 天回購率 | 判斷年度再行銷價值 |
| 回購曲線快速上升區間 | 找出最佳提醒時間帶 |

建議最後轉成：

| 客群 | 第一波提醒 | 第二波提醒 | 文案方向 |
| --- | --- | --- | --- |
| 高頻近期高價值客 | 上次投保後 45-60 天 | 90-120 天 | 強調快速續保與常旅保障 |
| 長天數高保費客 | 上次投保後 120-180 天 | 旅遊旺季前 | 強調完整保障與海外醫療 |
| 臨時投保短天數客 | 出發前 3 天情境 | 出發當天備忘 | 強調快速投保、少步驟 |
| 一次購買潛在回流客 | 連假/旺季前 | 年度旅遊檔期 | 強調再次出遊別忘記保障 |

實際提醒天數需依最終模型與 2025 驗證結果調整。

## 11. 行銷策略與文案產出格式

每個最終客群至少要有以下內容：

| 欄位 | 說明 |
| --- | --- |
| 客群名稱 | 用商業語言命名，例如「高頻近期高價值客」 |
| 客群輪廓 | RFM、旅遊型態、保費價值、購買習慣 |
| 2025 驗證結果 | 回購率、回購中位天數、KM 曲線重點 |
| 行銷優先級 | 高/中/低 |
| 提醒時機 | 第一波與第二波提醒時間 |
| 文案方向 | 文案主訴求 |
| 文案示範 | 1-2 則可放簡報的示範文字 |

文案示範不要太抽象，建議與客群特徵直接連動。

建議簡報頁面順序：

1. 封面：旅平險客群分群與回購提醒策略。

4. EDA 重點：客戶數、回購占比、保費價值、回購間隔。

6. 特徵設計：RFM 與候選特徵。
7. 特徵組合與 K 值比較。
8. PCA 3D 分群視覺化。

10. 2025 回購率驗證。
11. 各群回購速度/KM 曲線。

若頁數有限，可把特徵組合比較與模型細節放附錄，主線保留「問題、方法、結果、策略」。

## 13. 限制與風險

| 限制 | 說明 | 報告中應如何表述 |
| --- | --- | --- |
| 非因果分析 | 分群與回購差異是關聯，不是因果 | 不說「此群造成回購快」，改說「此群在 2025 呈現較快回購」 |
| 右設限 | 未在 2025 回購不代表永遠不回購 | 使用生存分析處理觀察期結束問題 |
| 時間窗差異 | 2023-2024 兩年、2025 一年 | 比較時要說明觀察窗不同 |
| K-means 假設 | K-means 偏好球狀群，對尺度敏感 | 必須標準化，並用 PCA/指標輔助檢查 |
| 特徵選擇主觀 | 不同特徵組合會得到不同分群 | 用特徵組合測試與商業解釋降低主觀性 |
| 付款代碼解釋限制 | `CARD_BANK` 對照銀行可能非官方 | 若使用，建議只作分群後描述，不作核心特徵 |

## 14. 實作檢查清單

1. 確認 2023-2024 特徵表不含任何 2025 欄位。
2. 確認每位 2023-2024 客戶都有 R/F/M。
3. 對高偏態欄位使用 `log1p` 或合理轉換。
4. 對進 K-means 的欄位標準化。
5. 跑 A-E 特徵組合與 K=2 到 K=6。
6. 輸出分群指標比較表。
7. 輸出 PCA 3D 圖。
8. 選出最終特徵組合與 K 值，並記錄原因。
9. 對各群做 2025 回購率與回購時間驗證。
10. 對各群產出策略與文案。

## 15. 暫定結論

本次修正版建議採用「RFM 為核心、少量行為特徵加值、2025 驗證」的分析方法。

此方法比舊版更適合期末報告，原因是：

1. 可以避免資料洩漏。
2. 特徵數較少，K-means 結果較可能清楚。
3. RFM 容易解釋，也容易轉成行銷策略。
4. 2025 驗證能讓報告更有說服力。
5. 生存分析可自然轉成提醒時機。

最後報告不要只說「分成幾群」，而要說：

> 我們如何用歷史投保行為找出可經營客群，並用 2025 回購表現驗證這些客群是否值得設計不同提醒策略。
